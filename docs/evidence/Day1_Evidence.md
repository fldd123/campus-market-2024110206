五、实验思考
1. 为什么课程不要求第一天立即开发页面  
因为软件工程的首要原则是先理解再动手。第一天聚焦于熟悉项目结构、技术栈、配置和业务流程，避免盲目编码导致返工。目录中 docs/ 空置也说明应先理解框架再填充内容。
2. "理解项目"比"开始编码"更重要  
- 降低返工率：不了解整体架构就写代码，后期重构成本极高
- 保持一致性：理解已有约定（别名、目录规范、代码风格）后才能写出符合项目的代码
- 规避风险：提前发现设计缺陷、依赖冲突、配置问题，而不是写到一半才发现
3. AI 在软件开发中的角色  
辅助者而非替代者——AI 擅长模式匹配、代码生成、搜索与解释，但理解业务、做架构决策、判断代码质量仍需人类开发者主导。AI 加速执行层，人把控设计层。
4. 今天最大收获  
理解了一个现代化 Vue 3 种子项目的完整骨架结构（api/components/router/stores/views 分层）、技术栈选型逻辑（Vite 8 + Pinia 3 + Vue Router 5），以及"先读后写"的工程思维——读代码的时间永远比写代码的时间有价值。
